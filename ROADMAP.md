# Дорожная Карта На 3 Месяца

План рассчитан на 5 занятий в неделю по 60-90 минут. Если времени меньше, лучше не сжимать темы, а двигаться медленнее.

## Лимит И Буфер Заданий

Чтобы курс не стал бесконечным, держим базовый маршрут: примерно 28 обязательных учебных issue/PR. Это не потолок, а скелет курса.

Дополнительно можно добавить 8-12 закрепляющих issue, если по ревью видно, что тема проседает: ввод, условия, циклы, Git, debugging, HTTP, SQL или тесты.

Ориентир по общему объему:

- минимум для прохождения курса - 28 core issue/PR;
- нормальный диапазон с закреплениями - 36-40 issue/PR;
- больше 40 issue заранее не создаем, чтобы не превращать курс в бесконечный список.

- Модуль 01: основы C# - 10 заданий.
- Модуль 02: инструменты, Git и debugging - 2 задания.
- Модуль 03: HTTP и API без базы - 4 задания.
- Модуль 04: SQL и PostgreSQL - 3 задания.
- Модуль 05: EF Core и PostgreSQL в API - 3 задания.
- Модуль 06: тесты и качество - 2 задания.
- Модуль 07: Docker и production habits - 1 задание.
- Финальный проект - 3 pull request.

Правило буфера: в активной работе держим только 2-4 ближайших issue. Остальные заранее созданные задания должны лежать в Backlog и не давить на темп.

Правило добавления extra issue: если брат не может объяснить решение, путается в базовой теме или делает задание только с подсказками, добавляем одно маленькое закрепляющее задание перед переходом дальше.

## Месяц 1: Основы Программирования И C#

Цель: брат умеет запускать консольные программы, читать простые ошибки, писать условия, циклы, методы и простые классы.

### Неделя 1: Старт, .NET, Терминал, Git

Темы:

- что такое программа;
- что такое C#, .NET SDK, проект, файл;
- `dotnet new`, `dotnet run`;
- `git status`, `git add`, `git commit`;
- ветка и pull request.

Практика:

- `Hello World`;
- `Hello Profile`: имя, возраст, проверки ввода.

Контроль:

- брат может сам запустить проект;
- может объяснить, что делает `dotnet run`;
- сделал минимум 3 коммита.

### Неделя 2: Типы, Строки, Условия

Темы:

- `string`, `int`, `bool`, `double`;
- `if`, `else`;
- `int.TryParse`;
- простая валидация.

Практика:

- калькулятор возраста;
- конвертер температур;
- проверка доступа по возрасту.

Контроль:

- брат понимает разницу между текстом и числом;
- умеет обработать неправильный ввод.

### Неделя 3: Циклы И Методы

Темы:

- `while`, `for`;
- методы, параметры, возвращаемые значения;
- разбиение большой задачи на маленькие.

Практика:

- игра "Угадай число";
- простой калькулятор с меню;
- повторение ввода до правильного значения.

Контроль:

- брат может вынести повторяющийся код в метод;
- объясняет, когда цикл остановится.

### Неделя 4: Коллекции

Темы:

- массивы;
- `List<T>`;
- `Dictionary<TKey, TValue>`;
- перебор коллекций.

Практика:

- список покупок;
- список контактов в памяти;
- поиск контакта по имени.
- подсчет слов через словарь.

Контроль:

- брат понимает, зачем нужна коллекция;
- умеет добавлять, искать и удалять элементы.

### Неделя 5: ООП Без Академизма

Темы:

- класс и объект;
- свойства;
- конструктор;
- список объектов;
- простая инкапсуляция.

Практика:

- `Contact` с именем, телефоном и email;
- консольная книга контактов;
- `TodoItem` со статусом выполнения.
- заметки с сохранением в файл.

Контроль:

- брат объясняет: класс - это свой тип данных;
- понимает разницу между классом `Contact` и объектом `new Contact(...)`.

## Месяц 2: HTTP, API И База Данных

Цель: брат понимает, что такое backend API, умеет сделать простые endpoints и начинает работать с SQL/PostgreSQL.

### Неделя 6: Инструменты И Debug

Темы:

- структура проекта;
- чтение compiler errors;
- debugger;
- NuGet;
- маленькие коммиты.

Практика:

- исправить намеренно сломанную программу;
- добавить NuGet-пакет;
- написать короткое описание PR.

Контроль:

- брат не пугается ошибок компилятора;
- может прочитать первую важную строку ошибки.

### Неделя 7: HTTP И JSON

Темы:

- client/server;
- request/response;
- HTTP methods: `GET`, `POST`, `PUT`, `DELETE`;
- status codes;
- JSON.

Практика:

- Minimal API с `/hello`;
- endpoint, который возвращает JSON;
- тест через браузер, HTTP file или Postman.

Контроль:

- брат объясняет, что происходит при `GET /todos/1`;
- понимает разницу между route, query string и body.

### Неделя 8: Todo API В Памяти

Темы:

- Minimal API или controller;
- endpoints;
- DTO на базовом уровне;
- валидация входных данных.

Практика:

- `GET /todos`;
- `GET /todos/{id}`;
- `POST /todos`;
- `PUT /todos/{id}`;
- `DELETE /todos/{id}`.

Контроль:

- API умеет CRUD без базы;
- неправильный ввод возвращает понятную ошибку.

### Неделя 9: SQL И PostgreSQL

Темы:

- таблица, строка, колонка;
- primary key;
- `SELECT`, `INSERT`, `UPDATE`, `DELETE`;
- `WHERE`, `ORDER BY`;
- простые связи.

Практика:

- таблица `todos`;
- таблица `subjects`;
- таблица `tasks`;
- несколько запросов руками.

Контроль:

- брат может написать простой `SELECT`;
- понимает, зачем нужен `Id`.

### Неделя 10: EF Core

Темы:

- entity-класс;
- `DbContext`;
- migrations;
- connection string;
- сохранение данных через EF Core.

Практика:

- Todo API с PostgreSQL;
- миграции;
- чтение и запись данных через API.

Контроль:

- API сохраняет данные после перезапуска;
- брат понимает, что EF Core не заменяет понимание SQL.

## Месяц 3: Качество, Docker И Финальный Проект

Цель: собрать маленький backend-проект, который можно показать и запустить с чистого места.

### Неделя 11: Тесты И Качество

Темы:

- unit test;
- integration test на базовом уровне;
- Arrange, Act, Assert;
- README;
- простая архитектура.

Практика:

- xUnit-тесты для методов;
- 3-5 тестов для API;
- улучшить валидацию.

Контроль:

- брат может написать failing test и сделать его passing;
- понимает, какое поведение защищает тест.

### Неделя 12: Финальный Проект

Вариант проекта: `Study Planner API`.

Функции:

- создать предмет;
- создать задачу по предмету;
- отметить задачу выполненной;
- получить список задач;
- фильтр по статусу или дате;
- PostgreSQL;
- EF Core migrations;
- валидация;
- тесты;
- Docker Compose;
- README с запуском.

Контроль:

- проект запускается по README;
- есть история коммитов;
- брат может устно пройтись по основным файлам.
