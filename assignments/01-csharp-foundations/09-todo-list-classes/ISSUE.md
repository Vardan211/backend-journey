# HW09: Todo List With Classes

## Goal

Build a console todo list where each task is a `TodoItem` object.

## Acceptance Criteria

- [ ] Runs with `dotnet run`
- [ ] Has a `TodoItem` class
- [ ] `TodoItem` has `Title` and `IsDone`
- [ ] Stores tasks in `List<TodoItem>`
- [ ] Shows the menu until the user chooses `0`
- [ ] Adds a task
- [ ] Handles empty task title
- [ ] Lists all tasks
- [ ] Lists only incomplete tasks
- [ ] Marks a task as done
- [ ] Deletes a task
- [ ] Handles unknown menu command
- [ ] Has at least 3 small commits
- [ ] PR includes reflection answers

## Reflection Questions

1. How is `TodoItem` similar to `Contact`?
2. What does the `IsDone` property mean?
3. Where does your code change a task's state?
4. How does the incomplete-task filter work?
5. What became clearer about OOP after the second class-based task?
