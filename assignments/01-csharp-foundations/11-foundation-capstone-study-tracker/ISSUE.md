# HW11: Foundation Capstone - Study Tracker Console

## Goal

Build a console study-task tracker that combines the main C# foundation topics: validation, loops, lists, dictionaries, methods, classes, and files.

## Acceptance Criteria

- [ ] Runs with `dotnet run`
- [ ] Has a `StudyTask` class
- [ ] `StudyTask` has `Title`, `Subject`, and `IsDone`
- [ ] Stores tasks in `List<StudyTask>`
- [ ] Shows the menu until the user chooses `0`
- [ ] Adds a task
- [ ] Handles empty task title
- [ ] Handles empty subject
- [ ] Lists all tasks
- [ ] Lists tasks by subject
- [ ] Marks a task as done
- [ ] Deletes a task
- [ ] Calculates subject statistics with `Dictionary<string, int>`
- [ ] Saves tasks to `study-tasks.txt`
- [ ] Loads tasks from `study-tasks.txt`
- [ ] Handles missing file without crashing
- [ ] Handles unknown menu command
- [ ] Has at least 6 actions extracted into methods
- [ ] Has at least 4 small commits
- [ ] PR includes reflection answers

## Reflection Questions

1. Which module 01 topics does this task use?
2. Where does your code create a `StudyTask` object?
3. Why does this program need `Dictionary<string, int>`?
4. Which methods did you extract and why?
5. How does the program save data between runs?
6. Which foundation topic still feels weakest?
