# HW08: Contact Book With Classes

## Goal

Build a console contact book where each contact is a `Contact` object with name, phone, and email.

## Acceptance Criteria

- [ ] Runs with `dotnet run`
- [ ] Has a `Contact` class
- [ ] `Contact` has `Name`, `Phone`, and `Email`
- [ ] Stores contacts in `List<Contact>`
- [ ] Shows the menu until the user chooses `0`
- [ ] Adds a contact
- [ ] Handles empty name
- [ ] Handles empty phone
- [ ] Lists all contacts
- [ ] Searches contact by name
- [ ] Deletes contact by name
- [ ] Handles unknown menu command
- [ ] Has at least 3 small commits
- [ ] PR includes reflection answers

## Reflection Questions

1. How is a `Contact` class different from a `string`?
2. Where does your code create a `Contact` object?
3. Why do you need `List<Contact>` instead of `List<string>`?
4. What happens if the user tries to delete a missing contact?
5. Which OOP idea is still most confusing?
